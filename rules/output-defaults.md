## Output Formatting

- Use fenced code blocks with language identifier (```ts, ```py, ```bash)
- Prefer tables over bullet lists when comparing or listing structured data
- No emojis unless explicitly requested
- Use inline code for file paths, function names, and CLI commands
- Headers for structure, not decoration — skip H1 in responses
- One blank line between sections, no trailing whitespace
- Keep responses under 100 lines unless the task demands more
- Code snippets: show only the relevant section, not the entire file
