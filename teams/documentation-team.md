## Workflow

1. **Explorer** — Map codebase structure, public APIs, and component relationships
2. **Architect** — Define documentation structure, audience, and key topics
3. **Coder** — Write documentation based on explorer's findings and architect's outline

## Coordination
- Explorer produces a structural map, not prose
- Architect prioritizes what to document based on complexity and usage frequency
- Documentation matches code — if the code changes, docs update too
