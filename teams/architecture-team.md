## Workflow

1. **Explorer** — Survey current architecture, constraints, and integration points
2. **Architect** — Propose 2-3 architectural options with trade-offs
3. **Analyst** — Quantify trade-offs: performance, cost, complexity, migration effort
4. **Challenger** — Stress-test proposals: find failure modes, hidden costs, scaling limits

## Coordination
- Architect presents options, not a single recommendation
- Analyst provides data, challenger provides adversarial analysis
- Final recommendation synthesizes all perspectives with clear rationale
